# 职教专业调研报告生成器

[![A级技能](https://img.shields.io/badge/Skill%20Grade-A-1a5276)](SKILL.md)
[![Process 模式](https://img.shields.io/badge/Pattern-Process-2471a3)](SKILL.md)
[![行数](https://img.shields.io/badge/Lines-378-2e86c1)](SKILL.md)

基于 AI Agent 的职业教育专业/专业群调研报告与可行性分析报告生成器。支持两种报告类型，输出 MD + HTML 双格式。

## 功能概述

- **双类型报告生成**：常规性专业调研报告（7章）+ 双高专业群可行性分析报告（10章）
- **智能信息收集**：5 维度关键词设计 + 网络搜索 + 知识库检索，自动整理 7 类素材
- **数据可信度管控**：六层内容分层（直接引用/衍生计算/推算标注/无数据/推断建议/公开推断），每个数字可追溯
- **措辞边界防护**：严格区分"事实（本校已开设）"与"推断建议（建议纳入·待校方确认）"，禁止假装调研
- **MD + HTML 双输出**：Markdown 源文件 + 自包含 HTML（内嵌 CSS，离线可用，自动生成目录）
- **搜索量闭环控制**：3 级失败处理 + 3 条件收敛标准 + 3 组上限，防止过度或不足搜索
- **双路径用户确认**：素材确认门（Phase 2.3）+ 大纲确认门（Phase 3.2），不可绕过
- **按需加载参考**：双高报告自动加载国家战略/产业链/组群框架，常规报告 Do NOT Load 双高专用文件

## 快速开始

### 作为 OpenCode Skill 使用

1. 将此仓库安装为 OpenCode skill
2. 对话中触发：`帮我调研一下XX专业` / `XX专业就业怎么样` / `专业群可行性`

### 生成报告

```bash
# HTML 转换（依赖 markdown + beautifulsoup4）
pip install markdown beautifulsoup4
python scripts/md_to_html.py --input "报告.md" --output "报告.html" --title "报告标题"
```

## 两种报告类型

| 类型 | 章数 | 适用场景 | 触发词 |
|------|------|----------|--------|
| **常规性专业调研报告** | 7章 | 单一专业调研、就业分析、课程优化 | "常规""调研""就业" |
| **双高专业群可行性分析报告** | 10章 | 双高申报、专业群建设可行性论证 | "双高""专业群""可行性" |

### 逻辑链

- **常规（7章）**：行业趋势 → 区域产业 → 岗位需求 → 人才供给 → 企业反馈 → 毕业生反馈 → 建设建议
- **双高（10章）**：国家战略 → 区域需求 → 产业微观 → 大数据分析 → 专业关联 → 组群逻辑 → 优势对比 → 可行性论证 → 建设建议

## 6 阶段工作流

| 阶段 | 核心动作 | 用户关卡 |
|------|----------|----------|
| 1 需求引导 | 4步配置（类型→标题→数据来源→区域） | — |
| 2 信息收集 | 5维度搜索→7类整理→素材确认 | 素材确认门 |
| 3 大纲规划 | 输出固定章节结构→大纲确认 | 大纲确认门 |
| 4 逐章撰写 | 按结构+素材顺序写 | — |
| 5 格式转换 | MD→HTML（内嵌CSS） | — |
| 6 多轮修改 | 定位章节→刷新HTML | — |

## 核心能力

- **六层数据可信度体系**：直接引用 → 衍生计算 → 推算标注 → 无数据支撑 → 推断建议 → 公开数据推断
- **双路径追问决策矩阵**：6 种用户输入场景的完整处理路径，区分"事实"与"推断建议"
- **搜索收敛标准**：3 条件 + 3 组上限，防止过度搜索和过早停止
- **7 类素材 → 章节映射表**：每类素材精确映射到常规/双高的对应章节
- **7 条 NEVER 规则 + 8 条 Do/Don't**：每条附带 WHY

## 内容规则优先级

| 优先级 | 规则 | 关键要求 |
|--------|------|----------|
| CRITICAL | 事实可信 | 每个数字有来源；衍生计算给出原始数据和公式 |
| HIGH | 结构完整 | 按类型章节不缺、逻辑链通顺、每章有结论 |
| MEDIUM | 信息密度 | 一段至少一个数字/案例，避免碎片化 |
| LOW | 格式规范 | 表格整洁、来源标注统一、HTML内联CSS |

## 准确性底线

- 每个事实性陈述必须能追溯到素材，无素材支撑不写入
- 禁止编造数据、禁止用"本次调研涵盖"假装实地调研
- 禁止将 agent 推断包装为学校事实（如"学校可能还开设"）
- 素材不足时标注"现有资料中该方向数据有限"，而非装作数据充分
- 岗位分析必须带职业分类代码（如 2-09-06-03）
