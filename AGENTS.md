# vocational-report-generator

OpenCode skill: 职业教育专业调研报告生成器。根据用户需求（专业/专业群）生成调研报告，输出 MD + HTML 双格式。

## 仓库结构

- `SKILL.md` — 指令文件，定义6阶段工作流。作为 skill 被加载时，这就是 agent 要执行的指令。
- `scripts/md_to_html.py` — Markdown 转独立 HTML（Python，依赖 `markdown` + `beautifulsoup4`）。
- `examples/数字媒体技术专业调研报告.md` — 完整报告范例，含数据表格、岗位分析表、职业分类表。
- `references/report-templates.md` — 两种报告类型的详细章节目录（常规7章/双高10章），撰写前必读。
- `references/feasibility-framework.md` — 双高报告专用框架（国家战略、产业链分析、组群逻辑等），**仅双高报告时加载**。
- `references/majors_report_structor.md` — 双高报告10章固定模板及每章内容要点，**仅双高报告时加载**。
- 无构建系统、无测试、无 CI、无 `opencode.json`。

## 两种报告类型（AGENTS.md 旧版只有7章，实际是双轨）

| 类型 | 章数 | 触发词 | 必需加载 |
|------|------|--------|----------|
| 常规性专业调研报告 | 7章 | "常规""调研""就业" | 无额外文件 |
| 双高专业群可行性分析报告 | 10章 | "双高""专业群""可行性" | `references/feasibility-framework.md` + `references/majors_report_structor.md` |

两种类型的结构和逻辑链不同，见 SKILL.md §3.1。

## 关键约束

- **所有事实必须追溯到素材**：无素材支撑的内容不写入报告；数字禁止二次运算，必须直接引用原始数据（推算需标注"据XX数据推算"）。犯规原因：平均值掩盖方差，可能导致培养方案决策错误。
- **素材不足标注**：某分类素材少于2条独立来源时，标注"现有资料中该方向数据有限"，用推断语气而非断言。
- **HTML 必须是独立自包含文件**（所有 CSS 内嵌，无 `<link rel="stylesheet">`）。
- **章节结构固定**：按类型不可增减。常规7章逻辑链为"行业趋势→区域产业→岗位需求→人才供给→企业反馈→毕业生反馈→建设建议"。
- **全部内容为中文**。
- **用户确认门有2个**：素材确认（Phase 2.3）、大纲确认（Phase 3.2）。除非用户明确说"跳过"，否则不可绕过。详见 SKILL.md Phase 2-3。
- **组群专业来源必须在 Phase 1 明确**：双高报告的"核心专业 + 支撑专业"清单，要么由用户提供（事实路径，措辞用"本校已开设"），要么由 agent 推断（建议路径，措辞用"建议纳入"，首处标注"待校方确认"）。agent 推断的全篇措辞禁止用"学校可能还开设"等事实性措辞。用户提供 1 个专业 + 未指明核心时，必须追问，不可默认走推断路径。
- **调研数据来源必须在 Phase 1 明确**（常规报告）：第5/6章涉及调研数据，必须由用户提供（"本次调研"措辞）/ 公开数据推断（"据XX平台数据"措辞）/ 留空（"建议校方补充"）三选一。禁止编造百分比却无来源标注。

## 数值规范

- 直接引用原始数据，不进行二次运算（如需推算，保留原始数字并标注"据XX数据推算"）。
- 标注每个数字的来源和时间，如"（来源：XX报告, 2025）"。
- 衍生计算（如增长率、占比）必须同时给出原始数据和公式：格式为"2022年A、2024年B，年均增长X%（据(A-B)/A/2计算）"。
- 禁止根据少于3个数据点断言趋势。

## NEVER 规则（来自 SKILL.md §4.4，AI 容易犯规）

| 违规 | 原因 |
|------|------|
| 素材<2条来源时假装数据充分 | 决策者基于不完整信息做判断 |
| 只写"增长X%"而不给原始数据 | 无法追溯和验证 |
| 岗位分析不带职业分类代码（如2-09-06-03） | 教育部门以代码对接岗位标准 |
| 根据2个数据点断言趋势 | 趋势陈述不可信 |
| 用"学校可能还开设"措辞包装 agent 推断的组群专业 | 事实/推断边界模糊，校方误以为这是既有办学基础 |
| 用"本次调研涵盖…""据调研显示"措辞但实际未做调研，编造百分比 | 决策者误以为是实证调研结果，基于虚假数据调整培养方案 |
| 编造数据或用不确定信息断言 | 教育决策比没有信息更危险 |

## 工作方式

逐章用 `file_write`（首章）/ `file_edit`（后续）追加到 `.md` 文件，完成后用脚本转 `.html`。**不并行写多章**，按顺序逐章输出。

```bash
# HTML 转换（在工作区根目录执行）
python scripts/md_to_html.py --input "路径/报告名.md" --output "路径/报告名.html" --title "报告标题"
```

脚本依赖安装：`pip install markdown beautifulsoup4`

### 脚本行为注意

- 标题检测依赖中文序号模式（一、二、三 → 层级1；（一）（二）→ 层级2；1. 2. → 层级3），**不是标准 Markdown `#` 语法**。报告 MD 中必须用 `**一、章节名**` 这种粗体中文序号格式，而非 `# 一、章节名`。
- 脚本自动生成目录（TOC）和每章"↑ 返回目录"链接。
- 如脚本缺依赖，参照 SKILL.md Phase 5 中的手动回退方案。
